    **Gap 1 — Fix accepted_toast interpolation:**

    In `src/features/progression/hooks/useProgression.ts`, the `useAcceptSuggestion` hook's `mutationFn` accepts only a `suggestionId: string` and returns void. The `onSuccess` callback therefore has no access to the exercise name or weight needed for toast interpolation.

    Fix by changing the mutation to accept a context object and pass it through:

    1. Change `useAcceptSuggestion` to accept an object `{ suggestionId: string; exerciseName: string; suggestedWeight: number }` instead of just a string.
    2. The `mutationFn` only uses `suggestionId` for the RPC call — the other fields are carried through for the `onSuccess` callback.
    3. In `onSuccess`, the second argument to the callback is the variables object. Use it to interpolate the toast:
       ```typescript
       toast.success(
         t("progression.accepted_toast", {
           weight: variables.suggestedWeight,
           exercise: variables.exerciseName,
         })
       );
       ```
    4. Do NOT change the function signature for `useDismissSuggestion` — its toast (`dismissed_toast`) has no interpolation variables.

    Then update `ProgressionSuggestionCard.tsx` to pass the new object shape:
    - Change `acceptMutation.mutate(suggestion.id)` to:
      ```typescript
      acceptMutation.mutate({
        suggestionId: suggestion.id,
        exerciseName,
        suggestedWeight: suggestion.suggested_weight_kg,
      })
      ```

    **Gap 2 — Fix error key name mismatch:**

    The hooks reference `progression.accept_error` and `progression.dismiss_error`. The JSON files define `error_accept` and `error_dismiss`.

    Fix by renaming the JSON keys to match the hook code (hook conventions are more natural — verb_noun — and changing JSON is less risky than changing multiple hook references):

    In `src/locales/en/translation.json`, within the `"progression"` namespace:
    - Rename `"error_accept"` to `"accept_error"`
    - Rename `"error_dismiss"` to `"dismiss_error"`

    In `src/locales/es/translation.json`, within the `"progression"` namespace:
    - Rename `"error_accept"` to `"accept_error"`
    - Rename `"error_dismiss"` to `"dismiss_error"`

    Keep all values unchanged — only rename the keys.
