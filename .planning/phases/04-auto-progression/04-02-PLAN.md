   - `suggested_weight`: "Suggested" / "Sugerido"
   - `error_accept`: "Failed to accept suggestion" / "Error al aceptar sugerencia"
   - `error_dismiss`: "Failed to dismiss suggestion" / "Error al descartar sugerencia"

     - Show both `t('progression.current_weight')` and `t('progression.suggested_weight')` labels
     - Two action buttons: outline "Not Now" (dismiss) and primary "Increase Weight" (accept)

     - Stale detection: if `suggestion.plan_exercise?.prescribed_weight_kg !== suggestion.current_weight_kg`, show a muted warning text using `t('progression.stale_warning')` and disable the accept button
   - On accept success: toast fires from the hook (already implemented in plan 01)
   - On dismiss success: toast fires from the hook
   - Use confirmation dialog (AlertDialog from existing `src/components/ui/alert-dialog.tsx`) for accept action per Claude's discretion recommendation (confirmation dialog for irreversible weight changes on mobile)
