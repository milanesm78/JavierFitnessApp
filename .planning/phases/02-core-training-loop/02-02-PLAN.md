- Top-level `useFieldArray` for training_days
- Fields: plan name (Input), cycle length weeks (number Input, default 4)
- "Add Training Day" button that appends a new day with auto-generated label "Day {next letter}" (A, B, C...)
- Map over dayFields rendering TrainingDayCard for each
- Submit button at bottom
- CRITICAL: Use field.id as key (not array index) per React Hook Form docs
- CRITICAL: Use `useWatch()` NOT `watch()` for any reactive field observation (React 19 compatibility)

- All visible text uses t() from useTranslation() -- no hardcoded strings

**Create `src/features/plans/components/training-day-card.tsx`:**
Card component for a single training day within the plan form.
- Props: control, dayIndex, register, onRemove, errors
- Input for day_label (editable, e.g. "Day A - Push")
- Nested `useFieldArray` for exercises within this day: `name: \`training_days.${dayIndex}.exercises\``
- "Add Exercise" button opens ExercisePickerDialog
- Map over exerciseFields rendering ExerciseRow for each
- Remove day button (with confirmation if day has exercises)
- Exercise reordering via up/down arrow buttons (using useFieldArray `move`) -- per user discretion, chosen over drag-and-drop for mobile-first v1

**Create `src/features/plans/components/exercise-row.tsx`:**
Single exercise prescription row within a training day.
- Props: control, register, dayIndex, exerciseIndex, exerciseName, onRemove, onMoveUp, onMoveDown, canMoveUp, canMoveDown
- Display exercise name (read-only, from the selected exercise)
- Input fields: prescribed_sets (number, 1-10), prescribed_reps (number, 1-100), prescribed_weight_kg (DecimalInput, 0-500)
- Up/Down arrow buttons for reordering (lucide-react ChevronUp/ChevronDown icons, 48px touch targets)
- Remove button (Trash2 icon)
- Compact horizontal layout optimized for mobile: name on top row, sets/reps/weight on bottom row
- Use register() for simple fields, DecimalInput for weight

**Create `src/features/plans/components/exercise-picker-dialog.tsx`:**
Dialog to search and select an exercise from the existing exercise library.
- Uses shadcn/ui Dialog component
- Fetches exercises via useExercises() hook (from Phase 1)
- Search input filters exercises by name (client-side filter for the typical library size)
- Displays exercises as a list with name and YouTube thumbnail
- On select: calls onSelect callback with exercise_id, exercise name, and default values (sets: 3, reps: 10, weight: 0)
- Close dialog on selection
