  <name>Task 1: Create Phase 2 database migration with tables, RLS, RPC functions, and indexes</name>
  <files>supabase/migrations/00002_training_loop_schema.sql</files>
  <action>
Create SQL migration file `supabase/migrations/00002_training_loop_schema.sql` containing:

**Tables** (follow the exact schema from 02-RESEARCH.md Pattern 2):
1. `training_plans` -- id (UUID PK), client_id (FK profiles), plan_group_id (UUID NOT NULL), version (INT DEFAULT 1), name (TEXT NOT NULL), cycle_length_weeks (INT NOT NULL DEFAULT 4), status (TEXT CHECK IN draft/active/archived), created_at, activated_at, archived_at. Add UNIQUE INDEX `idx_one_active_plan_per_client` ON (client_id) WHERE status='active'.
2. `training_days` -- id (UUID PK), plan_id (FK training_plans), day_label (TEXT NOT NULL), day_order (INT NOT NULL), UNIQUE(plan_id, day_order).
3. `plan_exercises` -- id (UUID PK), training_day_id (FK training_days), exercise_id (FK exercises), exercise_order (INT NOT NULL), prescribed_sets (INT NOT NULL), prescribed_reps (INT NOT NULL), prescribed_weight_kg (DECIMAL(6,2) NOT NULL), UNIQUE(training_day_id, exercise_order).
4. `workout_sessions` -- id (UUID PK), client_id (FK profiles), training_day_id (FK training_days), started_at (TIMESTAMPTZ DEFAULT now()), completed_at (TIMESTAMPTZ NULL).
5. `workout_sets` -- id (UUID PK), session_id (FK workout_sessions), plan_exercise_id (FK plan_exercises), set_number (INT NOT NULL), weight_kg (DECIMAL(6,2) NOT NULL), reps (INT NOT NULL), created_at (TIMESTAMPTZ DEFAULT now()), UNIQUE(session_id, plan_exercise_id, set_number).

**RLS Policies** (follow 02-RESEARCH.md Pattern 5):
- Enable RLS on all 5 tables.
- Create `is_trainer()` helper function (SECURITY DEFINER, checks JWT user_role).
- CRITICAL: Always use `(SELECT auth.uid())` and `(SELECT auth.jwt())` wrapped in SELECT for initPlan optimization.
- training_plans: Clients SELECT own (client_id = auth.uid()); Trainer SELECT all, INSERT, UPDATE.
- training_days: Clients SELECT own (via plan ownership); Trainer ALL.
- plan_exercises: Clients SELECT own (via training_day -> plan ownership); Trainer ALL.
- workout_sessions: Clients ALL own (client_id = auth.uid()); Trainer SELECT all.
- workout_sets: Clients ALL own (via session ownership); Trainer SELECT all.
- Use separate policies per operation, NEVER use FOR ALL for mixed-role policies.

**RPC Functions:**
1. `create_plan_version(source_plan_id UUID) RETURNS UUID` -- Deep-copies a plan with all training_days and plan_exercises. Increments version, sets status='draft'. SECURITY DEFINER. (From 02-RESEARCH.md Pattern 2)
2. `activate_plan_version(new_plan_id UUID) RETURNS VOID` -- Archives current active plan for the same client, activates the new plan. Atomic transaction. SECURITY DEFINER. (From 02-RESEARCH.md code example)
3. `get_client_dashboard() RETURNS TABLE(...)` -- Returns client list with full_name, email, status (active/inactive/pending), has_active_plan, today_workout_status (completed/in_progress/not_started), last_workout_at. Uses LATERAL JOINs for efficiency. SECURITY DEFINER. (From 02-RESEARCH.md Pattern 6)

**Performance Indexes:**
- idx_training_plans_client_status ON training_plans(client_id, status)
- idx_training_days_plan_id ON training_days(plan_id)
- idx_plan_exercises_day_id ON plan_exercises(training_day_id)
- idx_workout_sessions_client_id ON workout_sessions(client_id)
- idx_workout_sessions_day_id ON workout_sessions(training_day_id)
- idx_workout_sessions_started_at ON workout_sessions(started_at)
- idx_workout_sets_session_id ON workout_sets(session_id)

- PlanFormValues type derived from Zod schema
- PlanStatus type: 'draft' | 'active' | 'archived'

**Create `src/features/plans/schemas.ts`:**
- Use `import { z } from "zod"` (Zod 4 is the npm default)
- exerciseSchema: exercise_id (uuid), order (int min 0), prescribed_sets (int 1-10), prescribed_reps (int 1-100), prescribed_weight_kg (number 0-500)
- trainingDaySchema: day_label (string 1-50), order (int min 0), exercises (array of exerciseSchema, min 1)
- planSchema: client_id (uuid), name (string 1-100), cycle_length_weeks (int 1-52), training_days (array of trainingDaySchema, min 1)

**Create `src/features/plans/hooks/usePlans.ts`:**
- planKeys factory: all, lists, list(clientId), details, detail(planId), active(clientId)
- useActivePlan(clientId) -- fetches active plan with nested training_days -> plan_exercises -> exercises, sorts by order. Uses Supabase nested select.
- useClientPlans(clientId) -- fetches all plan versions for a client, ordered by version desc.
- usePlanDetail(planId) -- fetches a single plan with full nested data.

**Create `src/features/plans/hooks/usePlanMutations.ts`:**
- useCreatePlan() -- inserts training_plan + training_days + plan_exercises in sequence (plan first, then days, then exercises per day). Invalidates planKeys.lists().
- useUpdateDraftPlan() -- updates a draft plan's metadata and replaces its days/exercises. Only works on draft plans.
- useCreatePlanVersion(sourcePlanId) -- calls supabase.rpc('create_plan_version'). Invalidates planKeys.list(clientId).
- useActivatePlan() -- calls supabase.rpc('activate_plan_version'). Invalidates planKeys.list(clientId) and planKeys.active(clientId).
- useDeleteDraftPlan() -- deletes a draft plan (CASCADE deletes days/exercises). Invalidates planKeys.list(clientId).

**Create `src/features/workouts/types.ts`:**
- WorkoutSession, WorkoutSet interfaces matching DB rows
- SessionWithSets type (session + nested sets with plan_exercise details)
- LastSessionValues type: Map<string, { weight_kg: number; reps: number }[]>

**Create `src/features/workouts/schemas.ts`:**
- setLogSchema: plan_exercise_id (uuid), set_number (int min 1), weight_kg (number min 0), reps (int min 0)

**Create `src/features/workouts/hooks/useWorkouts.ts`:**
- workoutKeys factory: all, sessions(clientId), session(sessionId), sets(sessionId), lastSession(clientId, dayId), history(clientId)
- useWorkoutHistory(clientId) -- fetches past completed sessions with all logged sets, ordered by started_at desc. Default limit 20.
- useLastSessionValues(clientId, trainingDayId) -- fetches the most recent completed session for this training day, returns a Map of plan_exercise_id -> set values. This is the pre-fill source (locked decision: last session actual values, not prescribed).
- useSessionSets(sessionId) -- fetches all sets for an active session.

**Create `src/features/workouts/hooks/useWorkoutMutations.ts`:**
- useStartSession() -- inserts workout_session. Invalidates workoutKeys.sessions(clientId).
- useLogSet(sessionId) -- inserts workout_set with optimistic update (follow 02-RESEARCH.md TanStack Query optimistic update pattern). Uses onMutate/onError/onSettled pattern.
- useCompleteSession() -- updates workout_session.completed_at. Invalidates workoutKeys.sessions(clientId) and dashboardKeys.clients().

**Create `src/features/dashboard/types.ts`:**
- ClientDashboardRow interface: client_id, full_name, email, status, has_active_plan, today_workout_status, last_workout_at

**Create `src/features/dashboard/hooks/useDashboard.ts`:**
- dashboardKeys factory: all, clients, clientDetail(clientId)
- useClientDashboard() -- calls supabase.rpc('get_client_dashboard'). staleTime: 2 minutes.
- useClientDetail(clientId) -- fetches client profile + active plan + recent sessions for the client detail view.

**Create `src/lib/utils/youtube.ts`:**
- extractYouTubeVideoId(url: string): string | null -- Robust parser handling 4+ YouTube URL formats (watch, youtu.be, embed, /v/). From 02-RESEARCH.md Pattern 4.

**Create `src/lib/utils/cycle.ts`:**
