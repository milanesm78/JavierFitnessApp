       **Tables:**
       - `training_plans`: id (UUID PK), client_id (FK profiles), plan_group_id (UUID NOT NULL), version (INT DEFAULT 1), name (TEXT NOT NULL), cycle_length_weeks (INT DEFAULT 4), status (TEXT CHECK IN 'draft','active','archived'), created_at (TIMESTAMPTZ DEFAULT now()), activated_at (TIMESTAMPTZ), archived_at (TIMESTAMPTZ). Add unique partial index: `idx_one_active_plan_per_client ON training_plans (client_id) WHERE status = 'active'`.
       - `training_days`: id (UUID PK), plan_id (FK training_plans), day_label (TEXT NOT NULL), day_order (INT NOT NULL), UNIQUE(plan_id, day_order).
       - `plan_exercises`: id (UUID PK), training_day_id (FK training_days), exercise_id (FK exercises), exercise_order (INT NOT NULL), prescribed_sets (INT NOT NULL), prescribed_reps (INT NOT NULL), prescribed_weight_kg (DECIMAL(6,2) NOT NULL), UNIQUE(training_day_id, exercise_order).
       - `workout_sessions`: id (UUID PK), client_id (FK profiles), training_day_id (FK training_days), started_at (TIMESTAMPTZ DEFAULT now()), completed_at (TIMESTAMPTZ).
       - `workout_sets`: id (UUID PK), session_id (FK workout_sessions), plan_exercise_id (FK plan_exercises), set_number (INT NOT NULL), weight_kg (DECIMAL(6,2) NOT NULL), reps (INT NOT NULL), created_at (TIMESTAMPTZ DEFAULT now()), UNIQUE(session_id, plan_exercise_id, set_number).

       **Indexes** (from RESEARCH.md):
       - idx_training_plans_client_status ON training_plans(client_id, status)
       - idx_training_days_plan_id ON training_days(plan_id)
       - idx_plan_exercises_day_id ON plan_exercises(training_day_id)
       - idx_workout_sessions_client_id ON workout_sessions(client_id)
       - idx_workout_sessions_day_id ON workout_sessions(training_day_id)
       - idx_workout_sessions_started_at ON workout_sessions(started_at)
       - idx_workout_sets_session_id ON workout_sets(session_id)

       **RLS Policies** (from RESEARCH.md Pattern 5):
       - Enable RLS on all 5 tables
       - Create `is_trainer()` SECURITY DEFINER helper function returning `(select auth.jwt()->>'user_role') = 'trainer'`
       - ALWAYS use `(SELECT auth.uid())` wrapped in SELECT for initPlan caching optimization
       - training_plans: client SELECT own, trainer SELECT all, trainer INSERT, trainer UPDATE
       - training_days: client SELECT through plan ownership, trainer ALL
       - plan_exercises: client SELECT through day->plan ownership, trainer ALL
       - workout_sessions: client ALL on own, trainer SELECT all
       - workout_sets: client ALL through session ownership, trainer SELECT through session

       **RPC Functions:**
       - `create_plan_version(source_plan_id UUID) RETURNS UUID`: Deep-copies a plan (with all days and exercises) to create a new version with status='draft'. Increments version number. Uses SECURITY DEFINER. See RESEARCH.md Pattern 2 for exact implementation.
       - `activate_plan_version(new_plan_id UUID) RETURNS VOID`: Atomically archives current active plan and activates the new one. Uses SECURITY DEFINER. See RESEARCH.md code example.
       - `get_client_dashboard() RETURNS TABLE(...)`: Returns client list with activity status and today's workout status using LATERAL JOINs. Uses SECURITY DEFINER. See RESEARCH.md Pattern 6 for exact implementation.

       **Important:** Use `gen_random_uuid()` for UUID defaults (already available in PostgreSQL 13+/Supabase).
