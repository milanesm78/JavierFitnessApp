**Install Recharts:**
Run `pnpm add recharts` to add the charting library. This is the only new dependency for this plan.

**Chart helpers (src/features/progress/utils/chartHelpers.ts):**
- CHART_COLORS array: 5-6 visually distinct colors that work in both light and dark themes (use HSL values that have good contrast on both backgrounds, e.g., hsl(220, 70%, 50%) for blue, hsl(142, 70%, 45%) for green, etc.)
- toChartData(rows, field): Transforms measurement rows into { date: string, value: number }[] format for Recharts, filtering out null values.
- formatChartDate(dateStr, locale): Formats ISO date to short month+year using date-fns format() with locale support (es/en).
- MEASUREMENT_CHART_FIELDS: Array of { key, labelKey (i18n key), unit, category } for all chartable measurement fields. This drives the field selector UI.

**Strength progress hook (src/features/progress/hooks/useStrengthProgress.ts):**
- progressKeys query key factory: progressKeys.strength(clientId, exerciseId), progressKeys.measurements(clientId)
- useStrengthProgress(clientId, exerciseId, enabled?): Calls supabase.rpc('get_strength_progress', { p_client_id, p_exercise_id }). Returns data as { date: string, max_weight: number }[]. Only enabled when both clientId and exerciseId are truthy.
- Default date range: last 12 months (handled in the RPC function default args).

**StrengthChart (src/features/progress/components/StrengthChart.tsx):**
Props: clientId (string), exercises (array of { id, name } from exercise library).
- Exercise selector: dropdown/select (shadcn Select component) to pick which exercise to chart. Defaults to first exercise.
- Uses useStrengthProgress(clientId, selectedExerciseId) to fetch data.
- If data has fewer than 2 points, render ChartEmptyState with message "Log more workouts to see progress" (translated).
- If data has 2+ points, render Recharts LineChart inside ResponsiveContainer (height 300px):
  - XAxis with dataKey="date", tickFormatter using formatChartDate with current locale
  - YAxis with unit " kg"
  - CartesianGrid with strokeDasharray="3 3"
  - Tooltip with formatted date label and "kg" unit
  - Single Line (type="monotone", dataKey="max_weight", stroke from CHART_COLORS[0], dot with r=4)
  - XAxis interval="preserveStartEnd" to prevent label overlap on mobile
- Loading state: show spinner while data loads.
- Use useTranslation() for all labels. Use i18next's current language for date locale.

**MeasurementChart (src/features/progress/components/MeasurementChart.tsx):**
Props: clientId (string).
- Uses useClientMeasurements(clientId) from Plan 01's hooks to get all measurement history.
- Field selector: dropdown/select to pick which measurement field to chart. Uses MEASUREMENT_CHART_FIELDS for options, grouped by category in the selector. Defaults to "weight".
- Transforms data via toChartData(measurements, selectedField).
- Same chart pattern as StrengthChart: ResponsiveContainer, LineChart, XAxis with date formatter, YAxis with dynamic unit from selected field, CartesianGrid, Tooltip, Line.
- Empty state when fewer than 2 data points.
- Mobile-optimized: short date format, preserveStartEnd interval.
- Use theme-aware colors: read CSS custom properties if possible, or use CHART_COLORS that work on both light/dark.

**ChartEmptyState (src/features/progress/components/ChartEmptyState.tsx):**
Simple component: renders a Card with centered icon (TrendingUp from lucide-react, muted), title, and message. Props: message (string). Reusable for both chart types.

**MeasurementHistory (src/features/measurements/components/MeasurementHistory.tsx):**
Props: clientId (string), onNewMeasurement? (() => void, optional callback for the "Add" button).
- Uses useClientMeasurements(clientId) to fetch history.
- Renders a list of measurement entries as cards (mobile-first, not a table per user discretion for mobile readability).
- Each card shows: date (formatted with locale), weight, height, and a summary line showing how many optional fields were recorded (e.g., "6 skin folds, 3 diameters, 5 circumferences").
- Cards are sorted by measured_at DESC (most recent first).
- Expandable/collapsible detail: tapping a card shows all recorded measurements in that entry (use Collapsible from existing radix dependency).
- Show change deltas: if previous entry exists, show color-coded delta next to weight (green for decrease if context is weight loss, or neutral -- just show "+2.5 kg" or "-1.3 kg" in muted text).
- Empty state when no measurements exist: "No measurements recorded yet" with translated text.
- Optional header with "Add Measurement" button (visible when onNewMeasurement callback is provided -- used in trainer view).
- Loading state: skeleton cards.
